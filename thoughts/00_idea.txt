# Идея

Морф - это надстройка над языком, позволяющая расширять его по мере необходимости. Язык с такой надстройкой также будем называть морфом. Позволяя расширять язык, морф позволяет определить
полностью новый язык на базе исходного.

// В чём отличие от традиционного транслятора?
Обычно написание транслятора довольно хлопотное дело. // Почему?
Идея в том, чтобы сделать добавление новых конструкций максимально простым. Сделать добавление нового синтаксиса не намного сложнее, написания функций.

У нас три языка:
    1. Тот, с которого мы переводим (который определяем)
    2. На который переводим
    3. Тот, который используем для перевода

Макросы - частный случай:
    2 = 3 = base
    1 = расширение base
т.е. макросы просто расширение языка с помощью его самого

Поиск будущего ещё один частный случай:
    1 = запрос на типа естественном языке (на самом деле есть свои ключевые слова и правила)
    2 = условия/дерево запроса
    3 = любой ЯП, например, Javascript


# Элементы реализации

Из каких компонентов всё это дело состоит?
    - эти три языка
    - правила
    - система исполнения правил (возможно на 4м языке), но в простом случае на 3

Язык - это синтаксис плюс семантика. И то, и другое задаётся правилами. Правило состоит из матчащей части и функции развёртки. Синтаксис задаётся матчащей частью правила, семантика
определяется переводом на язык 2.
